「Pylith」简单教程(一)--Trelis建立网格模型

发表于 2018-08-04 | 分类于 Pylith | | 阅读次数:
Pylith是一个进行有限元模型计算的命令行软件,但并不包括网格模型的建立,需要其他软件诸如Lagrit,Trelis/Cubit等来完成,本文将以Pylith的example/3d/Tet4为例,介绍如何使用Trelis建立简单的网格模型.

一.Trelis界面

Trelis软件界面如下所示.

经常使用的区域:

  1. Command Panal: 能在图形界面进行快捷的Geometry创建,分割,组合以及网格的划分.
  2. Command Line: 命令行的形式实现与前者同样以及更多的操作.
  3. 工具栏: 前五个依次为文件的新建,打开,保存,导入和导出.中间包括运行journal脚本文件以及调整观察方式.选中方式等
  4. Model Tree: 模型各个部分均在此以body-volume-surface-curve-vertex的层级显示.每个元素都有对应的ID号以便指定.
    其余各个子界面的使用说明可见官方教程,再次不多赘述.注意Trelis的每一步操作都有对应的脚本命令,方便保存,查看以及分享,非常方便.

2. 分割长方体为上下两部分

在此我们创建一个平面分割长方体,上部分之后作为弹性层,下部分之后作为粘弹性层,并创建一个断层面分割弹性层方便之后分别进行计算.



更详细的分割过程可见官方教程.(需科学上网)

3. imprint & merge

在创建分割完geometry后,各个volume相接的点线面很可能并不是同一个,因此需要进行合并,这在复杂的geometry下会耗费一定的时间.

三.模型网格划分

我们可以指定各个curve,surface或者volume的划分单元大小然后再进行网格划分.volume的网格划分方案默认为map,可使用help volume scheme或者通过command Panal查看所有的网格划分方案,在此采用tetmesh划分方案,可按需选择.



完成网格划分后,建议进行网格质量检查,通常最小值要大于0.2才能够使Pylith进行快速稳定的迭代运算.可通

四.声明body以及边界面

之后我们需要声明block以及一些边界面,供之后Pylith计算及确定边界条件时调用.

五.导出网格文件

至此网格文件全部建立完毕,只需要以exo格式导出了.

从Command Line的输出可以看到你生成的网格文件有多少个元素以及节点.通常来讲,节点越多则计算时间越长.
